# FDL1 — FIDELIO Content Publishing Pipeline

Automated content publishing pipeline for FIDELIO Productions. Drop a video file, get drafts on all platforms.

## How it works

1. Editor drops `.mp4` in the client's watch folder
2. Watcher validates the filename and triggers the pipeline
3. Video uploads to YouTube (Unlisted), gets transcribed, captions generated by Claude
4. Drafts created in Publer for all active platforms
5. Human schedules publish dates in Publer
6. Webhook receiver updates the asset record when content goes live

## Setup

### Prerequisites

- Python 3.11+
- n8n instance (for production orchestration)
- Railway account (for webhook receiver)

### Install dependencies

```bash
# Watcher + scripts
pip install watchdog

# Webhook receiver
cd webhook-receiver
pip install -r requirements.txt
```

### Environment variables

Copy `.env.example` and fill in real values:

```bash
cp .env.example .env
```

See `.env.example` for all required keys.

### Run the watcher (mock mode)

```bash
cd scripts
FDL1_MOCK_MODE=true python watcher.py dre-alexandra
```

### Run end-to-end tests (mock mode)

```bash
cd scripts
FDL1_MOCK_MODE=true python test_e2e.py
```

### Deploy webhook receiver to Railway

```bash
cd webhook-receiver
railway up
```
