Fidelio Pipeline

Editor Setup Guide

Seven steps to get your local pipeline running. Each step includes a Claude Code prompt — paste it into VS Code and Claude will walk you through it.

macOS 12+ · Python 3.11+ · Homebrew + ffmpeg · Git · Claude Code CLI · DaVinci Resolve
Before You Begin

API Keys You'll Need

Get these six keys in hand before starting. Robert sends you the Google credentials file and the Blotato key — the rest you create yourself on each platform.

OPENAI_API_KEY
OpenAI — Whisper transcription platform.openai.com → API keys Create a new secret key. Billing must be active.
ANTHROPIC_API_KEY
Anthropic — Claude metadata generation console.anthropic.com → API keys Create a new key. Add credits to your account first.
FAL_KEY
fal.ai — AI thumbnail generation fal.ai/dashboard → Keys Format: key-id:secret — two parts separated by a colon.
GOOGLE_SERVICE_ACCOUNT_JSON
Google Sheets access Robert sends you this file. Save it somewhere permanent (not the Desktop), then set this env var to the full file path.
BLOTATO_API_KEY
Blotato — social scheduling (Dre's account) Robert sends you this key. Do not share it.
FLASK_SECRET_KEY
Local — Flask session secret Any random string. Generate one: python3 -c "import secrets; print(secrets.token_hex(32))"
How It Works

Your Machine vs. What's Shared

Each editor runs their own independent pipeline. You don't need anyone else online to process clips — everything happens on your Mac. Google Sheets is the shared record that everyone can see after each run.

Your Mac Only
  • File watcher (Pass 1 & 2)
  • Monitor UI — localhost:5401
  • Review UI — localhost:5400
  • .env + API keys
  • credentials.json
  • Footage & export folders
Shared
  • Google Sheets (pipeline log)
  • Blotato (Dre's social accounts)
  • FAL thumbnails (public CDN)
Updating the code: When Robert pushes a change to GitHub, run git pull in your repo folder and restart the daemon. Your local config (folder paths, .env) is yours — never pushed, never overwritten.

Step 01

Clone the Repository

Get the Fidelio pipeline code onto your machine. You'll need Git installed. The repo is private — log in with the shared mokshatools@gmail.com GitHub account (get the password from Robert).

Claude Code Prompt
I'm setting up the Fidelio post-production pipeline on my Mac. I need to clone the private GitHub repository.

The repo is: https://github.com/mokshatools-team/fidelio-pipeline
GitHub login: mokshatools@gmail.com (ask Robert for the password)

Please:
1. Check that Git is installed — run: git --version (if missing, install via Homebrew: brew install git)
2. Help me choose where to clone it (suggest ~/Documents/ if I'm unsure)
3. Run: git clone https://github.com/mokshatools-team/fidelio-pipeline.git
   — when prompted for GitHub username/password, use mokshatools@gmail.com + the password Robert gave me
   — if it asks for a "personal access token" instead of password, let me know and we'll handle that
4. Confirm the folder structure looks right — I should see: local/, web/, config/, setup.sh, onboarding.html
5. Check Python 3.11+ is installed — run: python3 --version (if missing or too old: brew install python@3.11)

Step 02

Run Setup

One script installs everything — Python dependencies, virtual environment, and cache directories.

Claude Code Prompt
I've cloned the Fidelio pipeline repo. Now I need to run the setup script.

The repo is at: [PASTE YOUR REPO PATH HERE — e.g. ~/Documents/fidelio-pipeline]

Please:
1. Run: bash setup.sh from inside the repo folder
2. Tell me if any errors appear during install
3. Confirm the venv/ folder was created
4. Show me that the key dependencies installed (requests, flask, openai, anthropic, google-auth)

If setup.sh fails, read the error and fix it before continuing.

Step 05

Set Your Watch Folders

The config ships with placeholder paths — you must replace them with your own local folders before anything runs. Tell Claude where your raw footage drops and where your finished exports land.

Claude Code Prompt
I need to set my local folder paths in the Fidelio client config before I can run the pipeline.

Open config/clients/dre-alexandra.json — it has placeholder values like "/Users/YOUR_USERNAME/path/to/your/ingest-folder" that need to be replaced with my real paths.

Please:
1. Open config/clients/dre-alexandra.json and show me the current placeholder values
2. Ask me where I want my ingest folder (raw footage drops here — Pass 1 picks it up)
3. Ask me where I want my exports/watch folder (finished edited exports drop here — Pass 2 picks them up)
4. Update ingest_folder, watch_folder, watch_folder_long, and watch_folder_short with my real paths
5. Save the file
6. Confirm those folders exist on disk — if not, create them
