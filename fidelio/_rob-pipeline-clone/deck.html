The Big Idea

Two folders.
Everything else is automatic.

You drop a raw clip into one folder. You drop a finished export into another. The pipeline does the rest — transcription, metadata, thumbnails, client sign-off, and scheduled posting.

Your only job Move files. Review the output. Hit Approve.
2
Full Pipeline

Seven stages.
One continuous flow.

📁 Raw Clip Drops into ingest folder
🎙 Pass 1 Whisper transcribes · Sheets logged
🎬 Edit Editor cuts in DaVinci Resolve
✂️ Pass 2 Export drops · Claude writes · FAL generates
🖥 Review UI Editor approves titles, thumbnail, schedule
👁 Client Page Client watches, signs off before it goes live
📤 Posted Blotato publishes to YouTube · IG · TikTok
Stage 1 of 7 — Pass 1

The raw clip lands.

  • 1 File watcher detects the clip — the moment it appears in your ingest folder, Pass 1 starts automatically
  • 2 Whisper transcribes the audio — OpenAI's speech-to-text runs locally on the file. Supports French and English
  • 3 A row is written to Google Sheets — clip name, date, transcript summary. Everyone can see it immediately in the shared Sheet
What you do Drop the raw clip. Watch the Monitor at localhost:5401 to see it process.
1
Stage 2 of 7 — Edit

You edit.
DaVinci does the rest.

This is the human step. Take the raw clip, cut it in DaVinci Resolve — trim, colour grade, add music, export. The pipeline waits.

  • 1 Open the clip in DaVinci Resolve — use the Pass 1 transcript as your edit reference if helpful
  • 2 Cut, grade, mix — full creative control. Nothing automated here. This is your work
  • 3 Export the final file — drop it into your watch folder. Pass 2 detects it automatically and picks up from there
✂
Stage 3 of 7 — Pass 2

The edited export lands.

  • 1 Whisper transcribes the final cut — a clean, complete transcript of the finished video
  • 2 Claude generates metadata — 3 title options, a full description, and tags. Written in the client's voice, informed by the transcript
  • 3 FAL generates the thumbnail — AI image generation using the client's brand references. Automatic first pass, editable in Review
  • 4 Video uploads to Dropbox — a direct streaming link is stored in Sheets so the client can watch and sign off
What you do Drop the finished export. Pass 2 runs in the background — usually 2–4 minutes.
Stage 4 of 7 — Monitor

Watch it happen in real time.

The Monitor is a local dashboard that shows every clip currently processing — what stage it's on, whether it succeeded, and any errors.

Open it at

localhost:5401

Starts automatically when you run the pipeline. No setup needed.

You'll see
  • Active processing stages
  • Progress per clip
  • Completion or error status
  • Live log stream
M
Stage 5 of 7 — Review UI

Your editorial desk.

Once Pass 2 completes, the clip appears in your Review UI. This is where you make every creative decision before anything is scheduled.

①
Pick a title
Claude gives you 3 options. Select the best one — or edit it.
②
Refine the thumbnail
Regenerate, adjust frame position, override the prompt entirely.
③
Review metadata
Description and tags are editable inline before scheduling.
④
Approve & schedule
Set a publish time. Choose platforms. Hit Approve.
Open it at localhost:5400/review/dre-alexandra
Stage 6 of 7 — Client Sign-Off

Before it goes live,
the client says yes.

A private, password-protected page where the client can watch each clip, approve it, or send it back with a note — on their phone, anywhere.

What they see
  • Thumbnail + title
  • Inline video player
  • Approve button
  • Request revision (with comment)
What happens after
  • Status updates in Sheets
  • Approved clips move to post queue
  • Revision notes visible to editor
C
Stage 7 of 7 — Publishing

Scheduled.
Posted. Done.

Approved clips are pushed to Blotato — the client's social scheduling tool — at the exact time you set in the Review UI.

▶
YouTube
Long-form episodes and Shorts. Title, description, tags, and thumbnail all pre-filled.
◈
Instagram
Reels and feed posts. Vertical thumbnails auto-detected.
◉
TikTok
Short-form vertical clips. Same metadata, platform-appropriate formatting.
Architecture

Fully autonomous.
No live sharing needed.

Each editor runs the complete pipeline independently on their own machine. There's no server to connect to, no live session to share.

Lives on your Mac only
  • File watcher (Pass 1 & 2)
  • Monitor UI — localhost:5401
  • Review UI — localhost:5400
  • Your footage & exports
  • .env + API keys
  • credentials.json
Shared by everyone
  • Google Sheets — pipeline log
  • Blotato — client post queue
  • FAL thumbnails — public CDN
  • Dropbox — client review videos
The Shared Log

Google Sheets is
the paper trail.

It's not a live feed between editors — it's a running record. Every clip that moves through the pipeline leaves a row. Useful for tracking, reviewing history, and knowing what's gone out.

Exports tab
  • File name + export date
  • Duration + content type
  • Transcript summary
  • Status (Processing → Ready → Approved → Posted)
Metadata tab
  • 3 title options
  • Description + tags
  • Thumbnail URL
  • Review URL (Dropbox link)
