# Video Post Studio V1 Shell Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the first functional Video Post Studio shell with shared core contracts and visible v1 modules for Import, TOA, and Transcript Chat.

**Architecture:** Use a small Flask-based modular monolith with route modules, service helpers, and contract-focused models. Keep persistence in memory for v1 so the shell and shared workspace contracts are proven before integrating TOA logic or external services.

**Tech Stack:** Python 3.9, Flask, unittest, dataclasses, Jinja templates
